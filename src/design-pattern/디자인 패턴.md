# What

소프트웨어 설계시 특정 상황에서 자주 만나는 문제를 해결하기 위해 사용할 수 있는 재사용 가능한 솔루션.

디자인 패턴은 주로 [[OOP]] 설계에 관한 것이고, 대부분 객체 지향적 설계 원칙을 이용해 문제를 해결한다.

# How

패턴의 설계 구조를 보면 대부분 비슷한데, 확장성 추구 방법이 대부분 두 가지 구조로 정리되기 때문이다.

하나는 클래스 [[상속]], 다른 하나는 오브젝트 [[합성]]이다.
